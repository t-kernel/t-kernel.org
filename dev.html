基础版本

这个版本的目的是，维护一个为移植到其它 ARM 芯片提供一个基础范本。以最小的修改，让 t-monitor 和 t-kernel 在 arm9 上运行。 获取 Open T-Kernel 代码库，并切换到 arm9 分支。 

$ git clone git://git.open.tkernel.org/t-kernel
$ git checkout arm9

修改代码

如果你的 Linux 机器下复制代码，又在 Windows 下修改代码，以下设置可以作为参考，避免文件权限问题。 

$ git config core.fileMode false

讨论交流

制作补丁

这个项目使用git做版本控制，可以用这个命令制作补丁。 

$ git format-patch -N1

代码

为了方便交流以及追溯，我们修改代码一般以这样的原则：一次修改只针对一个目的，一个目的尽量只有一次修改。
修改好之后，生成补丁，写上修改的目的以及说明。把补丁附带在邮件的末尾，发送到邮件列表。

开发流程

一些建议的原则

  • 一次修改只解决一个问题，一个问题尽量仅用一个补丁解决。
  • 引用代码尽量选择有活力项目作为来源，并尽量保持原样。
  • 修改代码时尽量保持代码风格与原来的代码相协调。
